Sem Docker:

1 - Com acesso a um Banco de Dados Postgres, crie um novo database

2 - Execute a sequência de queries disponíveis no arquivo bdini.sql para preparar a instância inicial do banco de Dados

3 - Se preferir, use o arquivo posting.backup para realizar um restore

4 - Modifique o arquivo postingConfig.js com as informações necessárias para conexão com o banco de dados de teste

5 - No mesmo diretório dos scripts de testes, rode o comando "npm run test"

Com Docker:

1 - No mesmo diretório do arquivo docker-compose.yml, execute o comando "docker-compose up -d"

2 - Configure o arquivo postingConfig.js (localhost, porta 5432, database postgres e senha postgres)

3 - No mesmo diretório dos scripts de testes, execute "npm run test"

4 - Para encerrar o banco de dados, volte ao diretório do arquivo docker-compose.yml e execute o comando "docker-compose down"
